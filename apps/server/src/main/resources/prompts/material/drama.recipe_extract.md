只输出 JSON，不要解释、不要 markdown 代码块。严格按下面结构：
{
  "title": "配方名（8 字内，概括打法，如「重生复仇·步步反杀」）",
  "summary": "一句话说明这套配方适合拍什么、爽点在哪",
  "mainline": "可复用的主线模板：保留结构与情绪曲线、爽点节奏，去掉原剧的具体人名/品牌/地名，换成占位描述",
  "beats": [
    { "no": 1, "hook": "本集钩子套路（类型化，如：开局打脸 / 身份反转）", "beat": "本集功能与情绪转折（通用化，不写原剧具体情节）" }
  ],
  "characters": [
    { "role": "key 或 extra", "archetype": "角色原型（如：扮猪吃虎的女主 / 笑面虎反派）", "desc": "性格与成长弧线套路" }
  ],
  "hooks": ["这套打法最关键的 3-5 个钩子 / 留扣手法"],
  "notes": "套用建议：节奏、单集时长、最适合的题材方向"
}

要求：
- 一定要「去具体化」：抽的是可迁移的结构 / 套路 / 爽点节奏，不是原剧的剧情梗概。任何人名、品牌、独有设定都要泛化成占位描述。
- beats 数量与原剧分集大纲相当（最多 12 条，超过则提炼成代表性的 12 条）。
- mainline 要能直接喂给「分集大纲 AI」当主线参考。
