## Anti-Patterns Verdict

**Does this look AI-generated? Yes — strongly.** It's the second-order AI tell: an AI tool that *aspires* to look serious but defaults to cyber-shimmer.

- **Self-incriminating comment** at [packages/ui/.../globals.css](packages/ui) — `/* Primary = neon cyan so default buttons align with the gradient CTA aesthetic */`. The system was *designed* to be neon. PRODUCT.md says the opposite.
- **Animated rainbow scrollbar** — `linear-gradient(180deg, rgba(6,182,212,.5), rgba(168,85,247,.5))` thumb in `globals.css`, explicitly commented "Cyberpunk Theme." This conflicts directly with PRODUCT.md's "Do not make this feel like a flashy AI demo: avoid neon gimmicks, sci-fi clichés."

The only absent absolute-ban is side-stripe borders.

## Overall Impression

The bones are right; the skin is the opposite of the brief. Shell architecture (provider, route-derived activeId, suspense fallback, persisted collapse, mobile drawer with z-stack notes) is grown-up work. But every surface-level decision (forced dark, neon-cyan primary, gradient text on the wordmark, glass cards, display-font card headings, 5-hue stat row, animated rainbow scrollbar) reads as "AI demo," which PRODUCT.md explicitly bans. The biggest single opportunity: **strip the neon and rebuild the token set**. Everything else is downstream.

## What's Working

1. **Shell architecture is sound.** `ProducerShellProvider` + `useProducerShell()` + route-derived `activeId` + `<React.Suspense>` fallback + localStorage-persisted collapse + mobile drawer with z-stack reasoning in comments. This is plumbing that holds up.
2. **Topbar shape is right.** Persistent wallet pill + unread dot + ⌘K search is exactly the ambient-state-plus-on-demand shape an operator console needs.
3. **Chinese-monolingual discipline is real.** `{ zh, en }` ternaries have been methodically purged and the legacy `translations.ts` is properly tombstoned with intent.

## Priority Issues

### [P0] Strip the neon aesthetic, rebuild the visual token set
- **Why it matters**: PRODUCT.md explicitly forbids "neon gimmicks, sci-fi clichés, novelty-first visual effects." The current `--primary: #06b6d4` + gradient text + gradient scrollbar + glass cards + `Sparkles` brand mark hit every banned tell. Operators trying to reconcile royalties read the surface as "creative AI demo" and discount the numbers.
- **Fix**: Pick one restrained primary (desaturated indigo or graphite), retire `bg-clip-text text-transparent` outside one designated marketing surface, replace the cyan→purple scrollbar, remove `Sparkles` as wordmark icon, rewrite the `globals.css` comment that encodes "neon by design." Keep semantic red/amber/green.
- **Suggested command**: `/impeccable distill apps/web-music` — strip-to-essence pass on tokens. Follow with `/impeccable colorize` to introduce a deliberate accent.

### [P0] Justify or kill forced dark mode
- **Why it matters**: `<html class="dark">` is hardcoded in [app/layout.tsx:30](apps/web-music/src/app/layout.tsx:30). No toggle, no `prefers-color-scheme`. The user scene ("MCN ops lead at 10am under office lights, 6h of approval queues") does not force dark; the choice reads as developer-preference, which compounds the AI-tool signal.
- **Fix**: Ship light as default, expose `next-themes` with system preference respect, keep dark as opt-in for monitoring/video flows that actually benefit.
- **Suggested command**: `/impeccable adapt apps/web-music` — covers theme strategy + responsive considerations.

### [P1] Display font in product UI
- **Why it matters**: `style={{ fontFamily: "var(--font-display)" }}` (Space Grotesk) is on landing H1, workspace logo, sidebar logo, *and every dashboard card heading and stat number*. Product register bans display fonts in UI labels, buttons, data. It undermines the "precise and quietly premium" target.
- **Fix**: Restrict Space Grotesk to landing H1 only (and even that is optional). Workspace logo + card titles → Inter 600. Drop the body font from 7 weights to 2.
- **Suggested command**: `/impeccable typeset apps/web-music`.

### [P1] Compress AgencyOverview from 9 sections to 3 with action-first framing
- **Why it matters**: Welcome → 5 stats → 3 charts → matrix → trend + top → 8 domains + AI suggestions → recent songs → activity feed. Nine decisions before scroll-end and nothing tells Wei what to do *first*. Fails PRODUCT.md principle #1 ("make the next action obvious"). Wei doesn't need a company portrait; she needs today's queue.
- **Fix**: (1) "需要我处理" — overdue approvals, pending royalties, expiring contracts with action buttons. (2) "本周走势" — one chart, one comparison. (3) "艺人状态" — roster matrix below the fold. Move the analytical strip to a dedicated `/analytics` route.
- **Suggested command**: `/impeccable shape apps/web-music/src/components/producer/dashboard` — re-plan IA before re-coding.

### [P2] Sidebar IA: dedupe artist destinations, fix dynamic labels, move artist switcher to topbar
- **Why it matters**: 艺人视图 / MCN与孵化 / AI艺人孵化 / AI形象锻造 / 造型与道具 are five artist-adjacent destinations in the first 6 slots with overlapping mental models. The workshop label renames itself based on `activeArtist.type`, which violates Nielsen #4. The artist switcher (a primary action) lives buried in the sidebar instead of the topbar where every CRM puts an account switcher. The "明星带货" group containing one item exposes unfinished IA.
- **Fix**: Collapse "艺人管理" to one route with sub-tabs; keep workshop renamed only in subtitle, not nav; promote artist switcher to the topbar; either fold "明星带货" into a section or wait until plural.
- **Suggested command**: `/impeccable distill apps/web-music/src/app/(workspace)/layout.tsx` — strip the duplicate destinations.

### [P2] Semantic color contract
- **Why it matters**: Red doubles as "system error" and "紧急 priority"; amber doubles as "warning" and "重要 priority"; cyan is brand + info + "建议". Two color systems collide in the same field of view.
- **Fix**: Reserve red/amber/green strictly for system state. Priority levels get neutral pills with weight contrast (P1/P2/P3), not hue.
- **Suggested command**: `/impeccable clarify apps/web-music` — fold color contract into the broader UX-copy + labels pass.

## Minor Observations

- `cyan-400` `Sparkles` is the literal brand mark in the workspace. Cyan-and-sparkle is the single most overused AI-tool signifier of 2024–26.
- Stat-card animation `delay: i * 0.08` cascades on every dashboard mount. Aggregate ~400ms of FOUC-by-design. Cut.
- "AI 增长建议" copy hard-codes mock strings ("铸造 '夏日' 勋章") in production. Either gate behind real API or label as sample.
- Sidebar logo `bg-clip-text` text breaks in any screenshot pasted into a Word doc — operators do this routinely. Slop signal compounds.
- Topbar wallet `bg-amber-500/10` + studio chip `from-cyan-500/30 to-purple-500/30` + red unread dot + brand-cyan sparkle = four chromatic events in 200px of horizontal space.
- `style={{ fontFamily: "var(--font-sans, 'Inter', sans-serif)" }}` coexists with `--font-sans` already set on `<html>`. Redundant.
- Inter is loaded with 7 weights (300/400/500/600/700/800/900). Two would do. Bundle bloat.

## Questions to Consider

1. If you stripped every gradient, every glass surface, every `font-display` heading, and every animated entrance from this dashboard, what's left that still says "AI Star Eco" — and is *that* the brand you want to own?
2. Wei is on her sixth login of the week and the welcome line still says "你好，制作人。这是今天的数据概览。" What changes about this page if you accept that the user has been here 200 times and only wants the queue?
3. The same `ProductLanding` component serves music, drama, and celebrity with only a gradient prop swap. If the three products are differentiated only by hue, are they three products — or one product wearing three costumes?
